对表单序列化（serialize）的一些联想

发表于 2016-11-25 | 分类于 编程

表单序列化

jQuery有两个方法 .serializeArray() 与.serialize().

  • .serializeArray()返回: Array

    描述: 将用作提交的表单元素的值编译成拥有name和value对象组成的数组。
    如[ { name: a value: 1 }, { name: b value: 2 },…]
    一般用于Ajax POST

  • .serialize()返回: String

    描述： 将用作提交的表单元素的值编译成字符串。
    如：single=Single&multiple=Multiple1&multiple=Multiple3&check=check2&radio=radio1
    一般用于GET

具体讨论serializeArray()与serialize()如下：

1. 对于简单的表单提交（不涉及到checkbox，select等），用serialize()问题不大

这时，序列化和反序列化都很简单

2. 但是如果表单在有多选的情况下，serialize()和serializeArray()显然都会出现问题

  • 对于serialize(),思路是对name相同的情况进行相应的判断，这里不细论；
  • 对于serializeArray()方法
    比如select,checkbox。此时返回的是：
    [{multiple:Multiple1},{multiple:Multiple3},{check:check1}},{check:check2}]
    而我们期望的是：
    [{multiple:[Multiple,Multiple3]},{check:[check1,check2]}]
    而且我们通过ajax提交的请求数据根对象期望是json对象而不是直接的json数组。
