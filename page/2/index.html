对表单序列化（serialize）的一些联想

发表于 2016-11-25 | 分类于 编程

表单序列化

jQuery有两个方法 .serializeArray() 与.serialize().

  • .serializeArray()返回: Array

    描述: 将用作提交的表单元素的值编译成拥有name和value对象组成的数组。
    如[ { name: a value: 1 }, { name: b value: 2 },…]
    一般用于Ajax POST

  • .serialize()返回: String

    描述： 将用作提交的表单元素的值编译成字符串。
    如：single=Single&multiple=Multiple1&multiple=Multiple3&check=check2&radio=radio1
    一般用于GET

具体讨论serializeArray()与serialize()如下：

1. 对于简单的表单提交（不涉及到checkbox，select等），用serialize()问题不大

这时，序列化和反序列化都很简单

2. 但是如果表单在有多选的情况下，serialize()和serializeArray()显然都会出现问题

  • 对于serialize(),思路是对name相同的情况进行相应的判断，这里不细论；
  • 对于serializeArray()方法
    比如select,checkbox。此时返回的是：
    [{multiple:Multiple1},{multiple:Multiple3},{check:check1}},{check:check2}]
    而我们期望的是：
    [{multiple:[Multiple,Multiple3]},{check:[check1,check2]}]
    而且我们通过ajax提交的请求数据根对象期望是json对象而不是直接的json数组。

HTTP状态码常用的状态码

一、200状态码：
成功2××： 成功处理了请求的状态码。
1、200 ：服务器已成功处理了请求并提供了请求的网页。
2、204： 服务器成功处理了请求，但没有返回任何内容。
二、300状态码：
重定向3×× ：每次请求中使用重定向不要超过 5 次。
1、301： 请求的网页已永久移动到新位置。当URLs发生变化时，使用301代码。搜索引擎索引中保存新的URL。
2、302： 请求的网页临时移动到新位置。搜索引擎索引中保存原来的URL。
三、400状态码：
客户端错误4×× ：表示请求可能出错，妨碍了服务器的处理。
400 (Bad Request/错误请求)
400 (SC_BAD_REQUEST)指出客户端请求中的语法错误。
401 (Unauthorized/未授权)
401 (SC_UNAUTHORIZED)表示客户端在授权头信息中没有有效的身份信息时访问受到密码保护的页面。这个响应必须包含一个WWW-Authenticate的授权信息头。例如，在本书4.5部分中的“Restricting Access to Web Pages./限制访问Web页。”
403 (Forbidden/禁止)
403 (SC_FORBIDDEN)的意思是除非拥有授权否则服务器拒绝提供所请求的资源。这个状态经常会由于服务器上的损坏文件或目录许可而引起。
404 (Not Found/未找到)
404 (SC_NOT_FOUND)状态每个网络程序员可能都遇到过，他告诉客户端所给的地址无法找到任何资源。它是表示“没有所访问页面”的标准方式。
四、500状态码：
服务器错误5×× ：表示服务器在处理请求时发生内部错误。这些错误可能是服务器本身的错误，而不是请求出错。
1、500 ：服务器遇到错误，无法完成请求。
2、503： 服务器目前无法使用（由于超载或停机维护）。
了解了http状态码在SEO中就比较好分析网站。

MVC模式笔记

发表于 2016-11-23 | 分类于 编程

百度百科对MVC的解释

MVC 是一种使用 MVC（Model View Controller 模型-视图-控制器）设计创建 Web 应用程序的模式:

  • Model（模型）表示应用程序核心（比如数据库记录列表）。
  • View（视图）显示数据（数据库记录）。

  • Controller（控制器）处理输入（写入数据库记录）。

  • Model（模型）是应用程序中用于处理应用程序数据逻辑的部分。
    　　 通常模型对象负责在数据库中存取数据。

  • View（视图）是应用程序中处理数据显示的部分。
    　　通常视图是依据模型数据创建的。
  • Controller（控制器）是应用程序中处理用户交互的部分。
    　　通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。

自己理解的MVC模式

MVC只是一种设计思想，并不是硬生生的三层模式，它体现的是逻辑上的一种低耦合的思想。比如，修改用户界面我并不需要修改相应的逻辑代码（即控制层），但便于理解，我将MVC用自然语言简单归纳如下：

  1. 视图层（即暴露给用户的接口），它可能是提供给用户的操作界面。
  2. 数据层（即数据库数据），也就是程序需要操作的数据或信息。
  3. 控制层（即用户在触发某个操作后对数据的处理过程），它负责根据用户从”视图层”输入的指令，选取”数据层”中的数据，然后对其进行相应的操作，产生最终结果。

由于flash的特殊性
